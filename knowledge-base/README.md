# Knowledge Base - Advocate Legal Assistant

This folder contains all documentation for the Advocate project. **Read this before making any changes.**

## Quick Start for Agents

1. Read `ONBOARDING.md` first - gives you context in 2 minutes
2. Check `TODO.md` - see what needs to be built
3. Follow `CODING_STANDARDS.md` - our conventions and patterns
4. Update `COMPLETED.md` after finishing any feature

## Files

| File | Purpose |
|------|---------|
| `ONBOARDING.md` | Quick start guide for new agents/devs |
| `PROJECT_OVERVIEW.md` | Product vision, MVP scope, user personas |
| `ARCHITECTURE.md` | Tech stack, folder structure, data flow |
| `COMPLETED.md` | Features that are built and working |
| `TODO.md` | Features pending implementation |
| `CODING_STANDARDS.md` | Code conventions, patterns, best practices |
| `API_REFERENCE.md` | API endpoints documentation |
| `DATA_SOURCES.md` | Legal data sources and how to add more |

## Updating This Knowledge Base

**Always update after:**
- Completing a feature → Update `COMPLETED.md` and `TODO.md`
- Adding new API → Update `API_REFERENCE.md`
- Adding new data → Update `DATA_SOURCES.md`
- Changing architecture → Update `ARCHITECTURE.md`

## Note

This folder is tracked in Git but not deployed (markdown files aren't served by Next.js).
